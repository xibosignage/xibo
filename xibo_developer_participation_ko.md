## Xibo 플랫폼 개발자 참여 방안

Xibo 플랫폼은 개발자나 디자이너가 디지털 사이니지 창작 과정에 참여할 수 있는 다양한 기회를 제공하며, 기술적인 요소를 네트워크에 추가하여 생동감 넘치는 결과물을 만들 수 있도록 지원합니다. 다음은 개발자가 Xibo 플랫폼에 참여할 수 있는 주요 방법과 각 방법에 대한 간략한 예시입니다.

| 참여 방안        | 설명                                                                                                                                                           | 예시                                                                                                |
|-----------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------|----------------------------------------------------------------------------------------------------|
| 위젯 (Widgets)    | 레이아웃 편집기를 사용하여 임베디드 또는 패키지 HTML 위젯을 만들거나, 자신만의 모듈을 개발하는 방법을 배웁니다.                                                                       | 레이아웃에 사용할 새로운 시각화 자료를 만듭니다. 예를 들어, 외부 데이터를 가져와 표시하는 위젯을 개발할 수 있습니다.                               |
| 플레이어 제어 (Player Control) | 플레이어 측에서 수행할 수 있는 작업, 위젯에서 사용할 수 있는 플레이어 정보, 실시간 데이터 표시, 재생 시간 제어, 웹훅 송수신 방법을 알아봅니다.                                                          | 플레이어에서의 상호작용 기능을 구현합니다. 예를 들어, 조명 센서를 통해 특정 콘텐츠를 트리거하는 기능을 만들 수 있습니다.                                |
| 테마 (Themes)     | CMS의 모양과 느낌을 변경합니다.                                                                                                                                     | CMS의 색상을 변경하거나 로고를 추가하여 CMS의 전체적인 외관을 맞춤 설정합니다.                                                              |
| 통합 (Integrate)  | Xibo API를 호출하여 외부 시스템을 연결합니다. oAuth를 통한 인증 방법과 사용 가능한 API 호출에 대해 배웁니다.                                                                      | 외부 시스템에서 Xibo로 콘텐츠를 푸시합니다. Xibo Canva 앱이 이러한 통합의 좋은 예시입니다.                                                       |
| 확장 (Extend)     | 추가 기능으로 CMS를 확장합니다. CMS 아키텍처와 미들웨어를 통해 자체 PHP 코드를 포함하는 방법을 배웁니다.                                                                           | 외부 시스템에서 콘텐츠를 가져오거나, 대시보드에 데이터를 다른 방식으로 표시하도록 CMS를 수정합니다. 외부 시스템이 호출할 수 있는 전용 API 경로를 작성합니다. |
| 플레이어 개발 (Creating a Player) | 특별한 사용 사례나 특정 하드웨어 요구 사항이 있거나 오픈소스 플레이어 중 하나에 기여하고 싶은 경우에 유용합니다. (주의: 상당한 노력이 필요할 수 있습니다.) | 플레이어가 CMS와 통신하고 콘텐츠를 렌더링하는 방법을 배웁니다.                                                                        |

### 오픈소스 관련 참고 사항

Xibo는 오픈소스이며, 특정 활동에는 몇 가지 시사점이 있습니다. 일반적으로 사용자 인터페이스를 통해 CMS에 코드를 입력하거나, 패키지 HTML로 업로드하거나, 웹훅을 보내거나, CMS에서 지원하는 XML 정의만을 사용하여 모듈/모듈 템플릿을 개발하거나, API를 통해 연결하는 경우 "독립적인(arms length)" 방식으로 작업하는 것으로 간주되며 원하는 대로 코드를 라이선스할 수 있습니다.

테마, PHP 클래스를 사용하는 사용자 정의 모듈, 커넥터를 개발하거나 Xibo를 확장하는 경우 해당 코드는 AGPLv3 라이선스의 적용을 받습니다.

### 기여 관련 참고 사항

핵심 소프트웨어에 포함되어야 한다고 생각되는 멋진 기능을 개발했다면 언제든지 환영합니다! 시작하기 전에 [기여 가이드라인](https://github.com/xibosignage/xibo/blob/master/CONTRIBUTING.md)을 참조할 수 있습니다.
